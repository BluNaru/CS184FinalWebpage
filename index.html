Our project is to generate an interactive solar system with unique planets. Like a real solar system, our simulation will include a star, rocky planets, and gaseous planets, with the planets orbiting around the star. The user can interact with specific planets to customize and explore the solar system.

So far, our group has an editable model of a rocky planet. The planet’s colors, size, and terrain can be edited manually. There are certain preset types of settings the user can apply to planets like the type of continents or land masses they want, or the color of the planet. Otherwise, the user can customize the planet to their liking by adjusting its size, land shape, and roughness to name a few parameters. If the user does not want to customize the planet, they can choose for our program to randomize the planet.

We also have a working orbit for the planets in the solar system. An assortment of rocky planets orbit around a star. The camera starts with a view of the entire solar system, which can zoom in and zoom out. In the planets’ orbit, the user can zoom in on a specific planet by selecting it from a dropdown menu. When zoomed in, the user can see what the planet looks like up close and possibly customize the planet as specified above. However, customization has not been added to the planets in orbit yet.

Gaseous planets are currently in progress, which when added, should be customizable similarly to the rocky planets. There is an option to add rings to gaseous planets, but we are still in the process of debugging the rings’ appearance. We will also give the gaseous planets a texture to simulate gas movement that we can see from pictures of real gaseous planets to make the planets less flat. Once gaseous planets are debugged, they will be added to the orbit and can be customized by the user.

In terms of our final product, planet customization and viewing planets in orbit have not been combined yet. For now, when viewing the orbit, the user can view each planet individually but cannot customize it yet. In a separate scene, the user can customize the planet but cannot see it in orbit at the same time. We will either have the user customize their planets before seeing them in orbit or add the option to customize their planet while they zoom in to a specific planet in orbit.

As of now, we still need to make sure all our components work together seamlessly. We only recently started incorporating the rocky planets into the orbit demo and created a working camera to easily view the solar system and specific planets. For gaseous planets, we need to debug the implementation of rings and apply a striped, gaseous texture to the planet itself so it is not a flat sphere. Once these components are completed and put together, we need to debug our code. At this point, the majority of our demo should be finished. With what we have completed now, we should be on schedule and can have a working solar system by the presentation deadline.
